COA - Basic Concepts

Components of Data Communication:

  • Message: The information or data to be communicated.
  • Sender: The device or person sending the message.
  • Receiver: The device or person receiving the message.
  • Transmission Medium: The physical or logical pathway through which the data travels.
  • Protocol: Rules governing data communication to ensure proper transmission.
  • Encoder/Decoder: Converts the message into a suitable format for transmission and vice versa.
  • Modem: Device for modulating and demodulating analog signals for digital data transmission.
  • Channel: The medium through which the signal travels (e.g., cables, airwaves).
  • Data Transmission Rate: The speed at which data is transmitted.
  • Error Detection and Correction: Techniques to identify and rectify transmission errors.

Distributed Processing:

  • Definition: The use of multiple interconnected computers to solve a problem.
  • Parallel Processing: Simultaneous processing of data by multiple processors.
  • Load Balancing: Distributing tasks among processors to optimize performance.
  • Fault Tolerance: Ability to continue processing despite hardware or software failures.
  • Interprocessor Communication: Methods for processors to exchange information.
  • Distributed Database: Data distributed across multiple locations or servers.
  • Client-Server Architecture: A model where tasks are divided between clients and servers.
  • Scalability: Capability to expand processing power by adding more resources.
  • Middleware: Software facilitating communication and data management in distributed systems.
  • Distributed Operating System: Software managing resources across a network.

Line Configuration, Topology:

  • Line Configuration: Physical arrangement of communication lines.
  • Point-to-Point Configuration: Direct communication between two devices.
  • Multipoint Configuration: Communication involving more than two devices.
  • Topology: The layout or structure of a network.
  • Bus Topology: Single central cable connecting all devices.
  • Star Topology: All devices connected to a central hub.
  • Ring Topology: Devices connected in a circular fashion.
  • Mesh Topology: Every device is connected to every other device.
  • Hybrid Topology: A combination of different topologies.
  • Advantages and Disadvantages: Evaluate the strengths and weaknesses of each configuration and topology.

Transmission Mode:

  • Simplex Mode: Communication is unidirectional (one-way).
  • Half-Duplex Mode: Communication is bidirectional but not simultaneous.
  • Full-Duplex Mode: Simultaneous bidirectional communication.
  • Examples: Walkie-talkies (half-duplex), Telephone conversation (full-duplex).
  • Bandwidth Usage: Full-duplex uses twice the bandwidth compared to half-duplex.
  • Delay: Full-duplex communication has lower delay compared to half-duplex.
  • Applications: Simplex for TV broadcasting, full-duplex for telephony.
  • Efficiency: Full-duplex is more efficient for continuous communication.
  • Collisions: Half-duplex can experience collisions in network communication.
  • Data Flow Control: Important in full-duplex to avoid overloading the receiver.

Categories of Networks:

  • Local Area Network (LAN): Limited geographic area, high data transfer rates.
  • Wide Area Network (WAN): Covers a broad area, lower data transfer rates.
  • Metropolitan Area Network (MAN): Intermediate size, connecting LANs in a city.
  • Personal Area Network (PAN): Small, for personal devices like smartphones.
  • Intranet: Private network within an organization.
  • Extranet: Extension of intranet, accessible to authorized external users.
  • Internet: Global network of interconnected networks.
  • Virtual Private Network (VPN): Secure communication over a public network.
  • Client-Server Network: Resources shared between clients and servers.
  • Peer-to-Peer Network: All devices have equal status, sharing resources.

OSI and TCP/IP Models:

  • OSI Model (Open Systems Interconnection): A conceptual framework with seven layers.
  • Physical Layer: Deals with the physical connection between devices.
  • Data Link Layer: Manages data framing, error detection, and MAC addresses.
  • Network Layer: Responsible for logical addressing and routing.
  • Transport Layer: Ensures end-to-end communication, flow control, and error recovery.
  • Session Layer: Establishes, maintains, and terminates sessions.
  • Presentation Layer: Translates data between the application and lower layers.
  • Application Layer: Provides network services directly to end-users.

Transmission Media: Guided and Unguided:

  • Guided Media: Signals travel through a physical medium.
  • Twisted Pair: Two insulated copper wires twisted together.
  • Coaxial Cable: Central conductor surrounded by insulation and a metallic shield.
  • Fiber-Optic Cable: Transmits data using light pulses through optical fibers.
  • Unguided Media: Signals travel through the air or space.
  • Radio Waves: Used in wireless communication, e.g., Wi-Fi.
  • Microwaves: High-frequency radio waves for point-to-point communication.
  • Infrared Waves: Short-range communication, e.g., TV remote.
  • Satellite Communication: Signals transmitted via satellites.
  • Advantages and Disadvantages: Evaluate the characteristics of each type of transmission medium.

Attenuation, Distortion, Noise, Throughput:

  • Attenuation: Weakening of a signal as it travels through a medium.
  • Causes of Attenuation: Resistance in conductors, distance traveled.
  • Distortion: Alteration of the signalâ€™s waveform during transmission.
  • Causes of Distortion: Signal reflections, echoes, interference.
  • Noise: Any unwanted signal that interferes with the desired signal.
  • Types of Noise: Thermal, crosstalk, impulse, electromagnetic interference.
  • Throughput: Actual data transfer rate achieved in a communication system.
  • Factors Affecting Throughput: Bandwidth, latency, protocol efficiency.
  • Bit Rate vs. Baud Rate: The difference between the number of bits and symbols transmitted per second.
  • Error Detection and Correction: Techniques used to identify and fix errors in transmitted data.

Propagation Speed and Time, Wavelength, Shannon Capacity:

  • Propagation Speed: Speed at which a signal travels through a medium.
  • Factors Affecting Propagation Speed: Type of medium, temperature.
  • Propagation Time: Time taken for a signal to travel from sender to receiver.
  • Wavelength: Distance between two consecutive points in a wave.
  • Relation Between Wavelength and Frequency: Inversely proportional.
  • Shannon Capacity (Channel Capacity): Maximum data rate a channel can handle.
  • Factors Influencing Channel Capacity: Bandwidth, noise level, signal-to-noise ratio.
  • Nyquist Theorem: The maximum data rate is twice the bandwidth of the channel.
  • Shannon's Theorem: Capacity increases with the signal-to-noise ratio.
  • Practical Implications: Understanding these concepts aids in designing efficient communication systems.
